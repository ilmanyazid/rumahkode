<h2><b>Apa itu CSS ?</b></h2>
<div>Pada tahun 1997 <a href="http://www.rumahkode.net/?elementor_library=landing-page-1">desain web</a> menjadi lebih mudah berkat diperkenalkannya Cascading Style Sheets (CSS). Awalnya dibuat untuk memisahkan konten dan struktur halaman web dari desain visual, CSS memungkinkan perancang membuat perubahan pada elemen visual ratusan halaman dengan mengubah satu atau beberapa baris kode. Tidak lagi seorang <a href="http://www.rumahkode.net/?elementor_library=landing-page-1">desainer web</a> harus memilah-milah setiap halaman untuk mengubah kode yang sama beberapa kali.</div>
<div></div>
<div>Selama bertahun-tahun, CSS telah berkembang dan versi sekarang mendukung:</div>
<ol>
 	<li>Font web</li>
 	<li>Animasi</li>
 	<li>Pidato</li>
 	<li>Tata letak yang fleksibel</li>
 	<li>Permintaan media yang memungkinkan situs web responsif</li>
 	<li>…dan banyak lagi.</li>
</ol>
<div></div>
<div>
<div>Gambar di bawah ini menunjukkan bagaimana CSS telah berkembang selama bertahun-tahun dari versi satu hingga versi ketiga CSS saat ini.</div>

<h4>5 Manfaat dari CSS</h4>
<div>Kami telah menyebutkan bahwa menggunakan CSS membantu meningkatkan efisiensi desain web.</div>
<div>Untuk mencapai ini, sebuah style sheet dibuat yang ada secara eksternal sebagai file terpisah, di dalam internal HTML antara tag &lt;head&gt;, atau inline dalam &lt;body&gt; dari HTML itu sendiri. Jadi, bagaimana cara menggunakan bantuan CSS?</div>
<div></div>
<div>1. Mempercepat proses perancangan web</div>
<div>Setiap pemilih CSS kemudian merujuk kembali ke lembar gaya untuk petunjuk tentang cara menampilkan elemen secara visual. Berkat ini, bila Anda ingin mengubah sesuatu secara universal di situs Anda, mengedit kode pada lembar gaya eksternal atau internal akan memastikan bahwa perubahan tersebut tercermin pada semua halaman.</div>
<div>Misalnya, jika Anda menggunakan file sheet gaya eksternal, dan Anda ingin mengubah pilihan font di situs Anda, Anda hanya perlu mengedit file CSS satu kali dan font baru ditampilkan di mana pun yang lama.</div>
<div></div>
<div>2. Membantu konsistensi</div>
<div>CSS tidak hanya membantu mempercepat pengembangan, namun juga memastikan konsistensi di semua halaman di situs Anda. Jika setiap halaman mengacu pada style sheet yang sama, semuanya akan terlihat dan bekerja sama.</div>
<div></div>
<div>3. Akses mudah</div>
<div>Menggunakan CSS juga mempermudah orang lain melakukan perubahan pada situs web. Jika Anda menyewa perancang baru untuk bekerja di situs Anda, mereka dapat melihat dan mengedit semua informasi styling dalam satu file CSS, daripada harus menelusuri dan mengedit puluhan atau bahkan ratusan halaman individual.</div>
<div></div>
<div>4. Lebih powerful dari HTML saja</div>
<div>CSS juga memberi Anda lebih banyak kemampuan presentasi daripada HTML. Menggunakan kemampuan &lt;div&gt;, misalnya, mengganti tabel.</div>
<div>Menetapkan persentase layar untuk elemen halaman memungkinkan fleksibilitas dalam presentasi blok konten pada halaman. Ada juga opsi pemformatan yang tersedia di CSS yang tidak disediakan HTML, seperti mengatur jarak antara karakter dan teks.</div>
<div></div>
<div>5. Banyak didukung</div>
<div>Akhirnya, CSS tidak hanya mudah dipelajari namun berhasil di semua browser utama. Dengan dukungan untuk Internet Explorer, Edge, Chrome, Firefox, Safari, dan Opera situs Anda akan terlihat sama untuk semua pengunjung.</div>

<h2>Apa itu CSS Framework?</h2>
<div> <img class="aligncenter size-full wp-image-509" src="http://www.rumahkode.net/wp-content/uploads/2018/02/belajar-css-framework.jpg" alt="belajar css framework" width="943" height="400" /></div>
<div>Dalam pengembangan perangkat lunak, mereka yang membuat aplikasi sering mengandalkan serangkaian kerangka kerja yang membantu standarisasi berbagai hal. Dalam desain web, kerangka kerja adalah paket yang terdiri dari file pra-bangun yang membuat desain web dan pengembangan front-end lebih cepat dan mudah.</div>
<div></div>
<div>Kerangka CSS sangat populer di kalangan perancang web karena mereka menyusun dasar situs untuk mereka. Kerangka kerja ini sering menyertakan kode yang menyediakan:</div>
<ol>
 	<li>Reset CSS untuk memastikan rendering browser yang berbeda dari HTML tidak akan menimpa stylesheet Anda</li>
 	<li>Tata letak grid untuk membantu desain web responsif</li>
 	<li>Elemen tipografi</li>
 	<li>Ikon</li>
 	<li>Styling untuk elemen visual seperti tombol, bentuk, dan tooltips</li>
 	<li>Banyak kerangka kerja mencakup elemen tambahan seperti fungsi JavaScript atau elemen GUI untuk membantu pengembangan aplikasi web front-end.</li>
</ol>
<div></div>
<div>3 Keuntungan Utama Kerangka CSS</div>
<div>Jika menggunakan style sheet untuk website Anda membantu mempercepat waktu desain, bayangkan apa yang memiliki semua elemen desain inti yang Anda buat? Betul. Ini membuatnya semakin mudah dan cepat untuk membangun konten web.</div>
<div></div>
<div>1. Perkembangan yang lebih cepat</div>
<div>Inilah sebabnya mengapa banyak desainer menggunakan kerangka kerja CSS untuk pekerjaan mereka. Dengan dasar-dasar yang sudah dijaga, semua yang perlu Anda lakukan adalah mengubah stylings visual untuk memenuhi kebutuhan Anda.</div>
<div></div>
<div>2. Mencoba dan menguji kode</div>
<div>Keuntungan lain menggunakan kerangka CSS adalah kenyataan bahwa Anda tahu kode itu bekerja. Ribuan desainer menggunakan kerangka kerja teratas, dan mereka telah menguji dan memverifikasi semuanya. Seharusnya tidak ada masalah dengan fungsionalitas atau kompatibilitas.</div>
<div></div>
<div>3. Ikuti praktik terbaik yang telah terbukti</div>
<div>Akhirnya, kerangka kerja CSS mendorong desain yang baik dan praktik pengkodean yang baik. Menggunakan grid mereka memudahkan pengorganisasian media dan konten di situs Anda dengan cara yang menarik secara visual. Mengacu kode terdokumentasi dengan baik membantu menanamkan praktik terbaik dalam pekerjaan Anda sendiri. Dan melihat apa yang terlihat seperti CSS bersih dapat membantu membuat Anda menjadi perancang yang lebih baik.</div>

Jika Anda berpikir menggunakan kerangka CSS yang tepat untuk Anda, maka Anda harus memutuskan mana yang akan digunakan; dan ada ratusan yang bisa dipilih.

<h3>Bootstrap</h3>
<img class="aligncenter size-full wp-image-510" src="http://www.rumahkode.net/wp-content/uploads/2018/02/belajar-css-dengan-bootstrap.jpg" alt="belajar css dengan bootstrap" width="646" height="523" />
Bootstrap, yang awalnya dikembangkan oleh Twitter pada tahun 2011, adalah kerangka kerja CSS yang paling populer dengan lebih dari 116.000 bintang di GitHub. Penggunaannya oleh Twitter untuk konsistensi di seluruh produk mereka bukanlah satu-satunya alasan keberhasilannya.

Dibangun dengan prinsip inti menjadi framework mobile-first, banyak perancang web mengubah Bootstrap untuk menghasilkan inti yang kuat untuk proyek desain web responsif mereka.
<ol>
 	<li>Beberapa kelebihan Bootstrap meliputi:</li>
 	<li>Responsiveness</li>
 	<li>Modularitas</li>
 	<li>Dimasukkannya template dan layout awal</li>
 	<li>Komunitas besar untuk dukungan dan pembelajaran</li>
 	<li>Sejumlah besar plugin pihak ketiga tersedia</li>
 	<li>Pada sisi negatifnya, Bootstrap's GUI Customizer adalah dasar. Misalnya, tidak termasuk pemetik warna sehingga</li>
 	<li>Anda perlu mengetahui nilai warna dan memasukkannya secara manual saat melakukan perubahan. Dokumentasi</li>
 	<li>untuk Bootstrap juga tidak sekuat beberapa yang lain dalam daftar ini.</li>
</ol>
<h3>Semantik UI</h3>
<img class="aligncenter size-full wp-image-511" src="http://www.rumahkode.net/wp-content/uploads/2018/02/belajar-css-dengan-semanticui.jpg" alt="belajar css dengan semanticui" width="942" height="503" />
Semantik UI CSS Framework dirilis pada tahun 2013, Semantic UI berfokus untuk menerapkan prinsip-prinsip bahasa alami ke dalam kerangka kerja agar kode lebih mudah dibaca dan dipahami. Dengan lebih dari 37.000 bintang di GitHub, ini adalah kerangka CSS terpopuler kedua yang tersedia.

Beberapa pro ke kerangka UI Semantic adalah:

Penggunaan ikon Ikon Awesome, walaupun beberapa perancang mungkin menganggap ini terlalu sering digunakan
Responsiveness
Modularitas
Pencantuman template starter
Dokumentasi yang bagus
Tutorial untuk menyesuaikan dan membuat tema
Sedangkan untuk kekurangan framework ini, tidak ada customizer GUI yang disertakan. Semua penyesuaian memerlukan pengeditan manual. Juga, tidak ada tambahan atau tambahan yang tersedia.
<h3>Foundation</h3>
<img class="aligncenter size-full wp-image-512" src="http://www.rumahkode.net/wp-content/uploads/2018/02/belajar-css-dengan-foundation.jpg" alt="" width="678" height="489" />

Foundation dari Zurb mengklaim, "kerangka front-end responsif paling maju di dunia." Dibangun pada tahun 2011, telah memiliki masa untuk matang menjadi kerangka kerja yang agak maju.

Kerangka ini menggabungkan beberapa aspek dari dua yang pertama dalam daftar ini: fokus bahasa semantik untuk markup bersih dengan mentalitas mobile pertama.

Dengan lebih dari 26.000 bintang GitHub, Foundation terletak di tengah paket dalam hal popularitas. Namun, ia menawarkan beberapa fitur bagus termasuk:

Tutorial dan pusat belajar yang sangat baik
Program sertifikasi untuk menunjukkan penguasaan
Tema starter bagus
Kerangka email untuk buletin, jaminan pemasaran, dan kebutuhan email lainnya
Ikon unik dari Yayasan
Bisnis terfokus
Responsiveness
Modularitas
Sementara Zurb's Foundation memang menawarkan sumber belajar yang sangat baik, dokumentasi tidak begitu kuat di situs mereka. Ada banyak sumber daya pihak ketiga yang tersedia sekalipun. Penyesuai GUI juga agak mendasar, sama dengan Bootstrap.
<h3>Pure
<img class="aligncenter size-full wp-image-515" src="http://www.rumahkode.net/wp-content/uploads/2018/02/belajar-css-dengan-pure.jpg" alt="belajar css dengan pure" width="675" height="424" /></h3>
Dibuat oleh Yahoo! Pada tahun 2013 Pure mengandalkan minimalisme sebagai fokusnya. Dengan lebih dari 17.000 bintang GitHub, kerangka ini sangat populer karena sangat ringan dan mendorong kustomisasi.

Untuk memberi gambaran bagaimana tulang-belulang Murni itu, tidak ada icon-set yang termasuk dalam framework. Anda dapat menggunakan Font Awesome jika Anda memilih, namun tidak ada yang terinstal sebelumnya dengan unduhan.

Sejauh menyangkut pro, penawaran Murni:

Dasar starter template
Ukuran file kecil
Responsiveness
Modularitas
Kerangka bersih bagi desainer yang hanya membutuhkan komponen tertentu
Dasar pembangun kulit GUI
Sementara beberapa melihat pendekatan minimalis sebagai keuntungan, pemula kemungkinan akan menganggap ini sebagai kelemahan. Kontra lain untuk Murni adalah dokumentasi, yang tidak terlalu kuat, dan fakta bahwa tidak ada add-on atau tambahan yang tersedia.

UIkit

<img class="aligncenter size-full wp-image-513" src="http://www.rumahkode.net/wp-content/uploads/2018/02/belajar-css-dengan-uikit.jpg" alt="" width="940" height="505" />
Kerangka CSS terakhir yang akan kita lihat mungkin hanya mendekati 11.000 bintang di GitHub, namun memiliki banyak penawaran. Dikembangkan pada tahun 2013 oleh YOOtheme, UIkit berfokus pada, "mengembangkan antarmuka web yang cepat dan canggih". Kebanyakan pemula akan menemukan ini tempat awal yang solid karena perpustakaan tema dan templat yang besar.

Pro lain yang ditawarkan UIkit meliputi:

Responsiveness
Modularitas
Sistem icon unik, menggunakan SVG, yang terus berkembang
Ekstra dan add-on yang tersedia, bagaimanapun, mereka mengenakan biaya untuk versi Pro
Penyesuai GUI canggih untuk tema dan template yang tersedia
Beberapa aspek negatif UIkit mencakup kebutuhan untuk membeli keanggotaan untuk akses ke fitur lanjutan dalam versi Pro. Bagi pemula yang bekerja dengan Joomla! atau WordPress ini bisa jadi pembelian yang bijak. Selain itu, dokumentasi tidak sekuat yang lainnya dalam daftar ini.
